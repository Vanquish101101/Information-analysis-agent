# 6. Дизайн (Фронтенд) — не применимо на MVP

У Агента 3 нет собственного пользовательского интерфейса. Он работает "под капотом": принимает
данные от Агента 1 и Агента 2, отдаёт результат через MCP-интерфейс (`analysis_digest`,
`analysis_detail`, `analysis_status` — см. `5. ТЗ.md` §3.2) и Telegram-уведомления при эскалации
(§3.3 того же файла). Пользователь не взаимодействует с Агентом 3 напрямую.

**На будущее:** возможен единый дашборд, объединяющий работу всех 4 агентов цепочки — но это
дизайн уровня Агента 1 (или отдельного координирующего слоя), не Агента 3. Если/когда такой
дашборд будет проектироваться, Агент 3 предоставляет для него данные через уже описанный
MCP-интерфейс — дополнительного дизайна с этой стороны не требуется.

Этот файл оставлен пустым по содержанию намеренно, чтобы сохранить нумерацию, единую для всех
4 агентов цепочки.
