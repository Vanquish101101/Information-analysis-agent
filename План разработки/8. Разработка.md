## 4. Рекомендуемый порядок разработки компонентов (MVP)

```
Шаг 1 — Инфраструктура
└── Docker Compose → подключение к marketing-agency-net → миграция Supabase (entities/claims/sources/runs/pending_user_decisions)

Шаг 2 — Чтение входа
└── Клиент чтения intelligence_agent.search_results (по telegram_id, created_at DESC)
    + чтение deep_parsing_agent.agent3_handoff_queue → нормализация в единый формат claim-кандидатов

Шаг 3 — Планировщик батча
└── Наблюдение за очередями с 08:00 → детекция тишины 15 мин → защитный потолок 11:00

Шаг 4 — Базовый граф LangGraph
└── Dispatcher → Send (параллельно) → Reducer — просто собрать claims без дедупа/сверки, проверить механику графа

Шаг 5 — Дедупликация
└── Gemini Embedding 2 → similarity ≥ 0.85 → LLM-judge подтверждение перед слиянием

Шаг 6 — Детекция противоречий
└── LLM-as-judge (структурированный вывод) + self-consistency (3 сэмпла) для спорных случаев

Шаг 7 — Контроль качества / эскалация
└── video-pipeline MCP клиент → порог $0.10 → лимит 1 автоповтор → cost cap $5/прогон →
    запись в pending_user_decisions при превышении

Шаг 8 — GlobalSynthesis
└── claude-sonnet-4-6 → дайджест (факты + confidence + количественные агрегаты)

Шаг 9 — MCP-сервер (выход)
└── analysis_digest / analysis_detail / analysis_status — полная реализация поверх сохранённых данных

Шаг 10 — Telegram-уведомления
└── Прямой вызов Bot API → эскалации дороже порога, cost cap, противоречия с устоявшимся фактом

Шаг 11 — Персистентность графа
└── LangGraph checkpoints → устойчивость к сбоям процесса
```

## 5. Версионирование и Git-конвенции

Те же конвенции, что у Агента 1 и Агента 2 — для единообразия в цепочке.

### 5.1 Стратегия веток

```
main          ← стабильная версия
dev           ← активная разработка
feature/xxx   ← отдельная фича
fix/xxx       ← исправление бага
```

### 5.2 Формат коммитов

```
feat: добавить узел дедупликации в граф анализа
fix: исправить порог cost cap в планировщике
refactor: вынести LLM-judge в отдельный модуль
docs: обновить схему MCP-интерфейса
chore: обновить зависимости
```
